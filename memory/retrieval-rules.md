# Retrieval Rules

## Retrieval Discipline
- Start with `memory/index.md`, then `memory/active-context.md`.
- Open `memory/requirements.md` for scope questions.
- Open `memory/decisions.md` for implementation choices and prior commitments.
- Do not scan every memory file by default.

## Update Discipline
- Write to the smallest relevant memory file.
- Do not repeat the same fact across multiple files unless it is necessary for navigation.
- Prefer short bullets over long narrative notes.

## When To Update
- Update `active-context` after milestones or focus changes.
- Update `requirements` when scope or constraints change.
- Update `decisions` when a choice is confirmed and should guide future work.

## When Not To Update
- Do not log temporary thoughts that do not affect future work.
- Do not create new memory files unless the existing structure becomes insufficient.
